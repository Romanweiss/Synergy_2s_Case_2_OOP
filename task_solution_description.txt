Подробное описание процесса решения задачи (разработка с нуля)
1. Анализ условия
- Исходное задание: написать тестовую программу, демонстрирующую работу методов базового и производного классов.
- Из условия определены обязательные элементы: базовый класс, производный класс и наглядный запуск методов.

2. Проектирование структуры классов
- Выбрана предметная область «сотрудники компании».
- Спроектированы два класса:
  - `Employee` как базовый;
  - `Developer` как производный.
- Выделены общие и специализированные методы, чтобы показать наследование и переопределение.

3. Реализация базового класса `Employee`
- Добавлены общие поля: `name` и `base_salary`.
- Реализованы методы:
  - `get_role()` — возвращает базовую роль;
  - `calc_salary()` — базовый расчёт зарплаты;
  - `info()` — единый формат вывода данных по сотруднику.

4. Реализация производного класса `Developer`
- Класс наследуется от `Employee`.
- В конструкторе используется `super()` для инициализации базовой части объекта.
- Добавлены поля, характерные для разработчика: `level` и `bonus`.
- Переопределены методы:
  - `get_role()` — возвращает роль разработчика с уровнем;
  - `calc_salary()` — считает зарплату с учётом бонуса.
- Добавлен собственный метод `write_code()`, которого нет в базовом классе.

5. Построение тестовой части программы
- В `main()` создаются объекты обоих классов.
- Выполняются прямые вызовы методов для демонстрации различий поведения.
- Реализована функция `print_staff(staff)`, принимающая список базового типа `Employee`.
- Через `print_staff` показан полиморфизм: одинаковый вызов `info()` приводит к разному поведению для `Employee` и `Developer`.

6. Проверка корректности
- Программа запускается командой `python main.py`.
- Проверяется, что:
  - методы базового класса работают корректно;
  - переопределённые методы производного класса вызываются корректно;
  - вывод полиморфной части демонстрирует разное поведение при едином интерфейсе.

Структура программы и ответственность компонентов
1. Класс `Employee`
- Хранит базовые данные сотрудника.
- Определяет стандартный интерфейс методов `get_role`, `calc_salary`, `info`.

2. Класс `Developer(Employee)`
- Расширяет и изменяет поведение базового класса.
- Демонстрирует наследование и override методов.

3. Функция `print_staff(staff)`
- Показывает полиморфную работу с объектами через базовый тип.
- Не зависит от конкретного подкласса, пока он поддерживает нужный интерфейс.

4. Функция `main()`
- Точка входа программы.
- Формирует тестовые объекты.
- Запускает демонстрацию прямых вызовов и полиморфного сценария.

Итог
Программа полностью соответствует условию задачи: с нуля реализована тестовая OOP-модель с базовым и производным классами и наглядной демонстрацией работы их методов.
