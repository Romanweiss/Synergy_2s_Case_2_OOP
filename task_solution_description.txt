Подробное описание процесса решения задачи (разработка с нуля)
1. Анализ условия
- Исходное задание: написать тестовую программу, демонстрирующую работу методов базового и производного классов.
- Из условия выделены обязательные требования: наличие базового класса, производного класса и наглядной демонстрации их методов.

2. Проектирование структуры
- Выбрана предметная область «сотрудники компании».
- Спроектированы классы:
  - `Employee` как базовый класс;
  - `Developer` как производный класс.
- Для демонстрации ООП предусмотрены наследование, переопределение методов и полиморфизм.

3. Реализация базового класса `Employee`
- Добавлены атрибуты `name` и `base_salary`.
- Реализованы методы:
  - `get_role()` — возвращает базовую роль;
  - `calc_salary()` — возвращает базовую зарплату;
  - `info()` — формирует унифицированную строку с информацией об объекте.

4. Реализация производного класса `Developer`
- Класс наследуется от `Employee`.
- В конструкторе используется `super()` для инициализации базовой части.
- Добавлены поля `level` и `bonus`.
- Переопределены методы:
  - `get_role()` — возвращает роль разработчика с уровнем;
  - `calc_salary()` — возвращает зарплату как `base_salary + bonus`.
- Добавлен метод `write_code()` как поведение, отсутствующее в базовом классе.

5. Построение тестовой демонстрации
- В `main()` создаётся один объект `Employee` и несколько объектов `Developer` (для разнообразия данных).
- Вывод разделён на отдельные блоки для удобства проверки:
  1) демонстрация базового класса;
  2) демонстрация производного класса;
  3) демонстрация полиморфизма.
- В каждом блоке явно подписаны вызовы методов и их результаты.

6. Демонстрация полиморфизма
- Реализована функция `print_staff(staff)` для списка базового типа `Employee`.
- В список передаются объекты разных классов (`Employee` и `Developer`).
- Один и тот же вызов `info()` показывает разное поведение за счёт переопределённых методов в производном классе.

7. Проверка корректности
- Программа запускается командой `python main.py`.
- Проверяется, что:
  - методы базового класса работают корректно;
  - переопределённые методы производного класса вызываются корректно;
  - полиморфный вывод через общий интерфейс работает для смешанного списка объектов.

Структура программы и ответственность компонентов
1. Класс `Employee`
- Хранит общие данные сотрудника.
- Определяет базовый интерфейс методов `get_role`, `calc_salary`, `info`.

2. Класс `Developer(Employee)`
- Расширяет и изменяет поведение базового класса.
- Демонстрирует наследование и переопределение методов.

3. Функция `print_staff(staff)`
- Выводит список сотрудников через единый интерфейс.
- Наглядно показывает полиморфизм для разных классов в одном списке.

4. Функция `print_header(title)`
- Формирует визуально выделенный заголовок секции.
- Делает вывод программы более очевидным для проверяющего.

5. Функция `main()`
- Точка входа программы.
- Создаёт тестовые объекты.
- Запускает пошаговую демонстрацию работы методов базового и производного классов.

Итог
Программа полностью соответствует условию задачи: с нуля реализована тестовая OOP-модель с базовым и производным классами, а вывод оформлен в явные логические блоки для удобной проверки результатов.
